## <a name="labor-cost-tracking-view"></a><span data-ttu-id="aabba-108">Lanaren kostuen jarraipenaren ikuspegia</span><span class="sxs-lookup"><span data-stu-id="aabba-108">Labor Cost Tracking view</span></span>

<span data-ttu-id="aabba-109">**Proiektuak** orrialdean, **Jarraipena** fitxa, hauta dezakezu **Jarraipena** > **Kostua** irekitzeko **Kostuen jarraipena** ikusi eta ikusi lan bakoitzaren gastuaren aurrerapena proiektuaren planean.</span><span class="sxs-lookup"><span data-stu-id="aabba-109">On the **Projects** page, on the **Tracking** tab, you can select **Tracking** > **Cost** to open the **Cost Tracking** view and see the progress of labor spend on each task in the project plan.</span></span> <span data-ttu-id="aabba-110">Ikuspegi honek zeregin batean gastatutako benetako lanaren kostuaren zereginaren aurreikusitako lanaren kostuarekin alderatzen ditu.</span><span class="sxs-lookup"><span data-stu-id="aabba-110">This view also compares the actual labor cost spent on a task to the task's planned labor cost.</span></span> <span data-ttu-id="aabba-111">Project Operations-ek formula hauek erabiltzen ditu lanaren kostuen metrikak kalkulatzeko:</span><span class="sxs-lookup"><span data-stu-id="aabba-111">Project Operations uses the following formulas to calculate labor cost metrics:</span></span>

<span data-ttu-id="aabba-122">Erakutsitako kostu kopuru guztiak proiektuaren kostuaren monetara bihurtzen dira kostuaren tasa zehazteko erabilitako proiektuaren kostuaren monetara.</span><span class="sxs-lookup"><span data-stu-id="aabba-122">All cost amounts shown are converted to the project's cost currency from the project's cost price currency used to determine the cost rate.</span></span> <span data-ttu-id="aabba-123">Proiektuaren kostu moneta proiektuan kontratatzeko unitatearen moneta da.</span><span class="sxs-lookup"><span data-stu-id="aabba-123">The cost currency of the project is the currency of the contracting unit on the project.</span></span> <span data-ttu-id="aabba-124">Urtean erakusten diren kostuen balio estimatuak **Aurrekontuak** fitxan **Proiektua** orrialdeak agian ez du aurreikusitako kostua gehituko **Jarraipena** fitxa. > - <span data-ttu-id="aabba-125">**Aurrekontuen fitxa** kalkulatutako kostua prezioen zerrendako kostu tasaren moneta bera erabiliz kalkulatzen du.</span><span class="sxs-lookup"><span data-stu-id="aabba-125">The **Estimates tab** calculates the estimated cost by using the same currency of the cost rate in the price list.</span></span> <span data-ttu-id="aabba-126">Orduan, prezio zerrendako monetan kalkulatutako kostua proiektuaren kostu monetan kalkulatutako kostu bihurtzen da.</span><span class="sxs-lookup"><span data-stu-id="aabba-126">Then, the estimated cost in the price list currency is converted to the estimated cost in the project's cost currency.</span></span> <span data-ttu-id="aabba-127">Proiektuaren monetan kalkulatutako kostua 2 hamartarrera biribilduta agertzen da.</span><span class="sxs-lookup"><span data-stu-id="aabba-127">The estimated cost in the project currency is shown rounded to 2 decimal places.</span></span> <span data-ttu-id="aabba-128">Bihurketa honen unean uneko moneta zehaztasuna ez da aplikatzen.</span><span class="sxs-lookup"><span data-stu-id="aabba-128">At no point during this conversion is currency precision applied.</span></span> 
> - <span data-ttu-id="aabba-129">**Jarraipena** fitxan, aurreikusitako kostuen kalkuluak zertxobait desberdina den kalkulu ordena jarraitzen du, moneta zehaztasuna bi etapatan aplikatzea eskatzen duena:</span><span class="sxs-lookup"><span data-stu-id="aabba-129">On the **Tracking** tab, the planned cost calculation follows a slightly different calculation order that involves currency precision being applied in two stages:</span></span>

   ><span data-ttu-id="aabba-132">Moneta zehaztasuna bi urratsetan aplikatzen da aurreikusitako kostua lortzeko (**Jarraipena** fitxa) zenbatetsitako kostutik (**Denbora mailakatua** ikuspegia **Aurrekontuak** fitxa).</span><span class="sxs-lookup"><span data-stu-id="aabba-132">Currency precision is applied in both steps to get a planned cost (on the **Tracking** tab) that deviates slightly from the estimated cost (on the **Time - phased** view on the **Estimates** tab).</span></span>
